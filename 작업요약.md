# 작업요약.md

## 현재 상태

Classic Map Blueprint v1 기준으로 기존 맵 구조를 유지하지 않고 Classic Map 전체를 재구축했다.

## 구현 완료

- 참가자 등록
- 공 자동 생성
- WINNER / LOSER 모드
- 참가자 이름 표시
- 실시간 TOP 5 순위판
- NEW LEADER 표시
- 상단 레이스 상황 HUD
- TOP5 평균 추적 카메라
- CAMERA ON / OFF 테스트 모드
- swept collision 기반 충돌 보강
- 동적 장애물 CCD 확장
- collision solver iteration
- 공 회전 상태 및 회전 마커
- 선두 공 물리 디버그 표시
- 템포 가드
- Classic Map Blueprint v1 기반 맵 구조
- START / A-B-C / MERGE / S COURSE / FAST-SAFE-CHAOS / BOTTLENECK / SCRAMBLE / FINISH 비율 구조
- 굵은 레이스 트랙 경로 시각화
- A / B / C 초반 루트
- FAST / SAFE / CHAOS 후반 루트
- MERGE ZONE
- BOTTLENECK
- SCRAMBLE

## 이번 작업 내용

- 기존 Sprint 2.8 맵 구조를 유지하지 않고 Blueprint 비율을 우선해 재구축했다.
- 전체 맵을 `START 10%`, `A/B/C 15%`, `MERGE 10%`, `S COURSE 20%`, `FAST/SAFE/CHAOS 20%`, `BOTTLENECK 10%`, `SCRAMBLE 10%`, `FINISH 5%` 흐름으로 재배치했다.
- 컨셉 이미지처럼 굵은 컬러 트랙 경로를 `TRACK_PATHS`로 추가했다.
- A/B/C와 FAST/SAFE/CHAOS 루트를 전체 폭의 80% 수준으로 명확히 분리했다.
- MERGE 구간을 하나로 합쳐지는 경쟁 구간으로 재설계했다.
- S COURSE는 좌우 이동이 4회 이상 발생하도록 하나의 긴 S 코스로 구성했다.
- BOTTLENECK은 TOP5가 한 화면에 모이도록 좁은 중앙 경로로 구성했다.
- SCRAMBLE은 결승 직전 최종 난전 구간으로 구성했다.
- Blueprint 금지 규칙에 맞춰 C자 함정을 제거했다.
- 동적 장애물은 스피너 1개만 유지했다.
- 해머와 이동 플랫폼은 이번 Blueprint 구현에서 제거했다.
- 장애물은 길을 막는 구조가 아니라 방향, 속도, 경로를 바꾸는 보조 요소로 축소했다.
- 화면 버전을 `Classic Blueprint v1`로 갱신했다.

## 주요 값

- visual ball size: `44px`
- physics ball radius: `16`
- gravity: `0.080`
- damping: `0.994`
- rolling resistance: `0.992`
- camera follow: `0.18`
- camera deadzone: `120`

## 남은 확인 사항

- 실제 브라우저에서 Blueprint 이미지처럼 레이스 트랙이 먼저 보이는지 확인이 필요하다.
- TOP5가 BOTTLENECK에서 한 화면에 모이는지 실제 플레이 확인이 필요하다.
- SCRAMBLE 구간에서 결승 직전 순위 변동이 충분한지 확인이 필요하다.
- 이미지 파일명이 `designclassic-map-concept-v1.png.png`로 되어 있어 추후 정리할 수 있다.
