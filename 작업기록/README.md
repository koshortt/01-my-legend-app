# 작업기록 폴더 운영 규칙

## 목적

`작업기록/` 폴더는 날짜별 작업 내용을 기록하기 위한 공간이다.

작업기록은 다음 목적을 가진다.

- 어떤 작업을 했는지 추적
- 어떤 파일을 수정했는지 확인
- 주요 결정 사항 기록
- 문제와 해결 과정 보존
- 다음 작업자가 맥락을 빠르게 파악

## 파일명 규칙

날짜별 Markdown 파일로 작성한다.

형식:

```text
YYYY-MM-DD.md
```

예시:

```text
2026-06-07.md
```

## 작성 시점

다음 경우 작업기록을 작성하거나 갱신한다.

- 기능 구현 완료
- 문서 작성 완료
- 버그 수정 완료
- 주요 구조 변경
- 중요한 의사결정 발생
- 작업이 중단되어 다음 작업자가 이어받아야 하는 경우

## 기본 작성 양식

```markdown
# YYYY-MM-DD 작업기록

## 작업 목표

- 

## 작업 내용

- 

## 수정 파일

- 

## 주요 결정 사항

- 

## 문제 및 해결

- 문제:
- 해결:

## 테스트 결과

- 

## 다음 작업

- 
```

## 작성 규칙

- 작업 내용은 간결하게 작성한다.
- 수정 파일은 실제 수정한 파일만 적는다.
- 해결하지 못한 문제는 숨기지 않고 기록한다.
- 불명확한 내용은 `추후 결정 필요`라고 표시한다.
- 작업기록은 과장하지 않고 사실 중심으로 작성한다.

## 금지 사항

- 다른 날짜 파일에 임의로 덮어쓰기 금지
- 실제로 하지 않은 작업 기록 금지
- 문제 발생 내역 삭제 금지
- 다른 프로젝트 작업 내용 기록 금지

## 보관 규칙

- 작업기록 파일은 삭제하지 않는다.
- 잘못된 기록은 삭제보다 정정 내용을 추가한다.
- 장기 보관 방식은 추후 결정 필요.
