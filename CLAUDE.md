@AGENTS.md

## 기본 작업 규칙

- 작업 전 `AGENTS.md`와 `프로젝트_기획서.md`를 먼저 확인한다.
- `다음_작업.md`가 있으면 현재 해야 할 작업을 확인한다.
- `작업요약.md`가 있으면 지금까지의 진행 흐름을 확인한다.
- `작업기록/` 폴더가 있으면 오늘 날짜의 작업기록 파일을 확인한다.
- 작업 대상 폴더 안에서만 파일을 수정한다.
- 다른 프로젝트 폴더는 수정하지 않는다.
- 요청받은 작업 범위를 넘지 않는다.
- 기존 기능과 화면 구조를 함부로 변경하지 않는다.
- 불필요한 리팩토링을 하지 않는다.
- 추정으로 기능을 만들지 않는다.
- 요구사항이 불명확하면 `추후 결정 필요`로 표시한다.
- 작업 완료 후 반드시 작업 내용을 문서에 반영한다.

---

## 문서 관리 규칙

### 프로젝트\_기획서.md

- 현재 프로젝트의 최종 기준 문서로 사용한다.
- 기능 추가 또는 변경 시 반드시 최신 상태로 유지한다.

### 개발\_로드맵.md

- 현재 개발 단계와 앞으로의 개발 계획을 관리한다.
- 완료된 항목은 체크 처리한다.

### 다음\_작업.md

- AI가 다음 작업에서 바로 수행해야 할 작업만 기록한다.
- 완료된 작업은 제거하고 다음 작업으로 갱신한다.

### 작업요약.md

- 지금까지 진행한 작업을 요약한다.
- 새 작업 완료 시 반드시 갱신한다.

### 작업기록/

- 날짜별 작업 기록을 저장한다.
- 파일명 예시:

```
작업기록/2026-06-07.md
```

- 작업 내용
- 수정 파일
- 주요 결정 사항
- 문제 및 해결 내용

을 기록한다.

## 개발 규칙

- MVP를 먼저 완성한다.
- 기능은 작은 단위로 구현한다.
- 한 번에 여러 기능을 추가하지 않는다.
- 구현 후 즉시 동작 확인이 가능한 상태를 유지한다.
- 오류 발생 시 원인 분석 후 수정한다.
- 임시 코드가 남지 않도록 한다.
- 하드코딩을 최소화한다.
- 설정값은 분리한다.

---

## UI 규칙

- 사용자가 이해하기 쉬운 UI를 우선한다.
- 화려함보다 사용성을 우선한다.
- 모바일과 PC 환경을 모두 고려한다.
- 일관된 디자인을 유지한다.

---

## 코드 품질 규칙

- 가독성을 우선한다.
- 함수는 하나의 역할만 담당하도록 작성한다.
- 중복 코드를 최소화한다.
- 파일 구조를 단순하게 유지한다.
- 주석은 필요한 경우에만 작성한다.

- 오류 없음
