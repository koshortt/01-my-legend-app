# 개발_로드맵.md

## 개발 원칙

- MVP를 먼저 완성한다.
- 사용자가 바로 실행하고 확인할 수 있는 단위로 개발한다.
- 레이스 감성, 관전 재미, 공정성을 우선한다.
- 불명확한 항목은 `추후 결정 필요`로 남긴다.

## Phase 0. 프로젝트 기준 정리

목표: 개발 기준 문서와 작업 운영 체계를 만든다.

- [ ] 프로젝트 기획서 작성
- [ ] AI 작업 규칙 작성
- [ ] 개발 로드맵 작성
- [ ] 다음 작업 문서 작성
- [ ] 작업 요약 문서 작성
- [ ] README 작성
- [ ] 작업기록 폴더 운영 규칙 작성

- [ ] WINNER / LOSER 모드 선택
- [ ] 기본 맵 구성

- [ ] 장애물 충돌

- [ ] 결과 화면
