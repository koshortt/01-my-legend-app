# 프로젝트\_기획서.md

## 프로젝트명

핀볼 레이스 (Pinball Race)

## 프로젝트 목표

핀볼 레이스는 단순한 추첨 게임이 아니라, 결과가 만들어지는 과정을 함께 관전하며 즐기는 `관전형 랜덤 레이스 게임`을 목표로 한다.

사용자는 버튼을 누르고 결과만 확인하는 것이 아니라 다음 과정을 즐긴다.

- 실시간 순위 경쟁
- 충돌
- 추월
- 역전
- 변수 발생
- 예상치 못한 결과
- 결승선 경쟁

최종 목표는 친구들과 함께 즐기는 랜덤 결정 플랫폼으로 성장하는 것이다.

## 프로젝트 비전

기존 룰렛, 사다리타기, 랜덤 추첨 서비스는 결과 중심이다.

핀볼 레이스는 결과보다 `과정`을 보여준다.

```text
참가자 등록
↓
공 자동 생성
↓
레이스 시작
↓
실시간 순위 경쟁
↓
랜덤 변수와 장애물
↓
역전과 결승 경쟁
↓
최종 결과 발표
```

## 한 줄 설명

룰렛보다 재미있는 관전형 랜덤 레이스

## 대상 사용자

### 1차 타겟

20~40대 친구들과 자주 모이는 사용자

사용처:

- 술자리
- 여행
- MT
- 회식
- 동호회

### 2차 타겟

스트리머, 인터넷 방송 진행자, 오픈채팅 이벤트 진행자

활용 예:

- 시청자 이벤트
- 벌칙 추첨
- 참여자 랜덤 선정

### 3차 타겟

유튜브 콘텐츠 제작자

활용 예:

- 랜덤 추첨 콘텐츠
- 벌칙 콘텐츠
- 팀 배정 콘텐츠

## 핵심 기능

### 참가자 등록

- 사용자는 참가자 이름과 공 개수를 입력한다.
- 예시:
  - `철수 x 5`
  - `영희 x 3`
  - `민수 x 10`

### 공 자동 생성

입력값을 바탕으로 공을 자동 생성한다.

예시:

- `철수-1`
- `철수-2`
- `철수-3`
- `영희-1`
- `민수-10`

### 레이스 진행

- 모든 공이 동시에 출발한다.
- 물리 기반으로 이동한다.
- 장애물과 충돌한다.
- 순위가 변동된다.
- 결승선 도착 순서를 기록한다.

### 실시간 관전 정보

- 실시간 순위판
- 순위 상승/하락 표시
- 선두 변경 표시
- 이벤트 로그
- 참가자 이름 표시

### 카메라 시스템

목표는 사용자가 가장 재미있는 장면을 보도록 하는 것이다.

우선순위:

1. 순위 급상승 공
2. 선두 경쟁 공
3. 결승선 근처 공

현재 MVP에서는 선두 추적 카메라부터 구현한다.

Auto Director Camera는 추후 확장한다.

### 맵 시스템

기본 맵:

- 클래식 핀볼

확장 맵 후보:

- 정글 맵
- 우주 맵
- 공장 맵

상세 내용:

- 정글 맵: 회전 통나무, 덩굴 장애물, 점프 패드
- 우주 맵: 저중력, 워프 포털, 이동 플랫폼
- 공장 맵: 컨베이어 벨트, 해머, 회전 기어

### 랜덤 이벤트 시스템

추후 구현 후보:

- 부스터
- 슬로우
- 점프 패드
- 순간 이동
- 폭탄
- 자석

구체 규칙은 추후 결정 필요.

### 결과 발표

- 최종 순위 집계
- TOP 3 발표
- 우승자 발표
- LOSER 모드일 경우 최하위 당첨자 발표

## 권한

MVP에서는 별도 권한 시스템을 사용하지 않는다.

MVP 제외:

- 로그인
- 계정 시스템
- 관리자 권한
- 서버 저장
- 결제

추후 플랫폼화 단계에서 필요한 권한:

- 사용자 계정
- 방 생성자 권한
- 방송 진행자 권한
- 관리자 권한

구체 권한 정책은 추후 결정 필요.

## 성공 기준

### 기능적 성공 기준

- 참가자 등록 가능
- 공 자동 생성 가능
- 레이스 시작 가능
- 물리 기반 레이스 진행 가능
- 장애물과 충돌 가능
- 순위 계산 가능
- WINNER 모드 결과 출력 가능
- LOSER 모드 결과 출력 가능
- 한 판이 오류 없이 종료됨

### 재미 기준

- 사용자가 결과보다 레이스 과정을 보기 위해 끝까지 시청한다.
- 추월과 역전이 관찰된다.
- 결승 직전 순위 변동이 가능하다.
- 친구들과 함께 웃고 즐길 수 있다.

### 관전 기준

- 누가 앞서는지 즉시 알 수 있다.
- 공과 참가자를 구분할 수 있다.
- 카메라가 중요한 장면을 놓치지 않는다.
- 순위 변동을 체감할 수 있다.
